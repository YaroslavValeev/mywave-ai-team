# Owner — протокол ежедневной работы (v1)

Короткий порядок дня без «архитектуры»: что делать, чтобы система давала пользу, а не шум.

## 1. Утро — проверка «можно ли работать»

Откройте экран миссии в Dashboard: вверху блок **«Как работать сегодня»** и **статус дня** (зелёный / жёлтый / красный).

**Красная зона — стоп.** Не планируйте новые шаги, пока не поправите записи. Красный, если уже есть данные, но они в плохом виде:

- меньше **70%** шагов с отметкой времени;
- меньше **50%** шагов с записанным итогом;
- меньше **80%** лидов с указанием источника;
- меньше **70%** сделок с суммой.

*(Если записей ещё нет, красный не включается «из воздуха».)*

**Жёлтая зона — осторожно.** Работать можно, но решения по «росту» проверяйте здравым смыслом: мало повторений, мало свежих шагов за неделю, или средний объём данных по типам шагов.

**Зелёная зона — можно планировать день.**

## 2. День — не больше трёх шагов

Выберите **1–3** шага, которые реально доведёте до конца. Лучше меньше, но с итогом.

## 3. Одна задача — одно действие — одно завершение

Не распыляйтесь: закончили шаг — сразу зафиксируйте статус и итог.

## 4. После шага — обязательная фиксация

Всегда:

- отметьте выполнение;
- укажите итог: **ничего** / **лид** / **продажа**;
- у продажи укажите **сумму**.

Без записи итога день для системы «пустой», подсказки деградируют.

## 5. Если подсказки «роста» странные

При жёлтой зоне или малом числе повторений **не усиливайте автоматически** альтернативный шаг — оставайтесь у базового плана, пока данные не окрепнут.

## 6. Вечер — минимальный контроль

За день должно быть хотя бы:

- **один** завершённый шаг;
- **один** записанный итог (даже «ничего»).

## Мини-KPI на день

- 1–3 шага начаты;
- ≥ 1 шаг завершён;
- ≥ 1 итог записан (если шаг был).
