# DOMAIN MODEL

Статус: draft v1  
Дата: 2026-04-07  
Назначение: единая модель сущностей и связей для Product/Governance/Runtime.

## 1) Project

- Обязательные поля: `project_id`, `slug`, `name`, `status`, `owner_id`, `created_at`, `updated_at`.
- Owner: SoT store (`shared-core` persistence owner).
- Lifecycle: `ACTIVE -> PAUSED -> ARCHIVED`.
- Связи: `Project 1:N Task`, `Project 1:N MemoryEntry`, `Project 1:N Artifact`.

## 2) Task

- Обязательные поля: `task_id`, `project_id`, `title`, `owner_text`, `status`, `domain`, `task_type`, `criticality`, `plan_or_execute`, `execute_gate`, `created_at`, `updated_at`.
- Owner: SoT store.
- Lifecycle: `NEW -> TRIAGED -> IN_PIPELINE -> IN_ROUNDTABLE -> IN_COURT -> WAIT_OWNER -> APPROVED_WAIT_MERGE|REWORK|NEED_INFO -> DONE|ARCHIVED`.
- Связи: `Task N:1 Project`, `Task 1:N Run`, `Task 1:N Decision`, `Task 1:N Approval`, `Task 1:N Artifact`, `Task 1:N ExecutionEvent`.

## 3) Run

- Обязательные поля: `run_id`, `task_id`, `orchestrator`, `status`, `phase`, `started_at`, `finished_at`, `triggered_by_channel`.
- Owner: Runtime Layer (`Molt`) с записью в SoT.
- Lifecycle: `QUEUED -> RUNNING -> STOPPING -> COMPLETED|FAILED|CANCELLED`.
- Связи: `Run N:1 Task`, `Run 1:N ExecutionEvent`, `Run 1:N Artifact`.

## 4) Decision

- Обязательные поля: `decision_id`, `task_id`, `run_id`, `decision_type`, `rationale`, `created_by`, `created_at`.
- Owner: Governance Layer (`Agents`) с записью в SoT.
- Lifecycle: immutable event stream.
- Связи: `Decision N:1 Task`, `Decision N:1 Run`, `Decision 1:N Approval`.

## 5) Approval

- Обязательные поля: `approval_id`, `task_id`, `decision_id`, `required`, `status`, `requested_at`, `resolved_at`, `resolved_by`.
- Owner: Governance Layer с записью в SoT.
- Lifecycle: `REQUESTED -> APPROVED|REJECTED|EXPIRED`.
- Связи: `Approval N:1 Task`, `Approval N:1 Decision`.

## 6) Artifact

- Обязательные поля: `artifact_id`, `task_id`, `run_id`, `kind`, `title`, `path`, `mime_type`, `checksum`, `created_at`.
- Owner: Runtime generates, SoT indexes metadata.
- Lifecycle: `CREATED -> INDEXED -> RETAINED|PURGED`.
- Связи: `Artifact N:1 Task`, `Artifact N:1 Run`, `Artifact N:1 Project`.

## 7) MemoryEntry

- Обязательные поля: `memory_id`, `project_id`, `scope`, `content`, `source_ref`, `created_at`.
- Owner: shared-core memory service.
- Lifecycle: append/update with retention policy.
- Связи: `MemoryEntry N:1 Project`, optional links to `Task`/`Artifact`.

## 8) Channel

- Обязательные поля: `channel_id`, `type`, `external_ref`, `capabilities`, `status`.
- Owner: shared-integrations registry.
- Lifecycle: `ACTIVE|DEGRADED|DISABLED`.
- Связи: referenced by `Run.triggered_by_channel`, `ExecutionEvent.channel_id`.

## 9) Agent

- Обязательные поля: `agent_id`, `role`, `layer`, `capabilities`, `policy_scope`, `status`.
- Owner: Governance/Runtime registries.
- Lifecycle: `ENABLED|DISABLED`.
- Связи: referenced in `ExecutionEvent.actor_id`, `Decision.created_by`.

## 10) PolicyRule

- Обязательные поля: `rule_id`, `scope`, `condition`, `effect`, `priority`, `enabled`.
- Owner: shared-policy package.
- Lifecycle: versioned rule set.
- Связи: applied to `Task`, `Run`, `Decision`, `Approval`.

## 11) ExecutionEvent

- Обязательные поля: `event_id`, `task_id`, `run_id`, `event_type`, `phase`, `payload`, `created_at`.
- Owner: Runtime writes; Governance reads/audits.
- Lifecycle: immutable event stream.
- Связи: `ExecutionEvent N:1 Task`, `ExecutionEvent N:1 Run`, optional `Channel`/`Agent` refs.

## 12) Owner Matrix (single ownership)

- SoT persistence owner: shared-core store.
- Orchestration owner: Runtime (`Molt`) for `Run` and execution phases.
- Governance owner: `Agents` for decisions/approvals/policy outcomes.
- Product owner: `Personal_Helper` for user-facing orchestration experience.

## 13) ID Policy

- `task_id`: generated by SoT storage only.
- `run_id`: generated by Runtime, persisted in SoT.
- `decision_id`, `approval_id`, `event_id`, `artifact_id`: generated by SoT.
