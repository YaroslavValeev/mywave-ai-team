# Контроль качества и безопасность (v1.1)

## 1) Основные риски
- Утечка секретов (.env, токены, ключи)
- Утечка PII (телефон, имя, Telegram ID)
- Галлюцинации и “уверенные” факты без доказательств
- Несогласованность решений между агентами
- Rework циклы без прогресса
- Выполнение критических действий без апрува Owner

## 2) Митигирующие меры
- Redaction в логах и Telegram (маскирование телефонов/токенов)
- Reality Checker обязателен для HIGH/CRITICAL
- Evidence-first: важные утверждения должны иметь доказательство (ссылка/лог/репро шаги)
- Max rework cycles = 2; 3-й цикл = варианты A/B/C на Owner
- Gate: EXECUTE CRITICAL/HIGH только после Owner approval

## 3) Human-in-the-loop (обязательные эскалации)
- Прод-деплой и миграции
- Публикации в крупные каналы/публичные обещания
- Ценообразование/платежи/договоры
- Любая обработка видео/фото/данных с идентификацией людей
- Удаления/деструктивные операции

## 4) Минимальные тест-сценарии (smoke)
- BUGFIX: воспроизведение → фикc → регресс
- CONTENT: источники → обработка → preview → publish approval
- EVENTS: правила → прозрачность → апелляции
- GAME: экономика → антиэксплойты → плейтест план
