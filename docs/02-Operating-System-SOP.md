# Operating System (SOP) — MyWave AI-TEAM v1.1

## 1) Роли (фиксировано)
- **Owner/Strategist**: стратегия + финальное утверждение. Канал итогов: Telegram (бот-личка).
- **AGM**: триаж, декомпозиция, skill-based routing, управление 3 стадиями, финальный вердикт.
- **Production Team**: агенты, работающие в Pipeline и Roundtable.

## 2) Принцип: Plan vs Execute
Система разделяет:
- **PLAN**: анализ, дизайн, планы, тексты, спецификации, pull-request план, сметы, сценарии.
- **EXECUTE**: реальное действие: деплой в прод, публикация в крупный канал, платеж/цена/договор, обработка персональных данных, удаление/миграция.

Правило: **EXECUTE по CRITICAL/High-impact — только после подтверждения Owner** (semi-autonomous).

## 3) Стадии выполнения
### Stage I — Pipeline (Sequential)
Цель: получить экспертный Draft v1 и комплект артефактов (структурно).
- 3–6 шагов, каждый шаг заканчивается **HANDOFF**.
- Автор решения **не может быть ревьюером** своего артефакта.

### Stage II — Roundtable (Peer Review)
Цель: независимая критика, поиск конфликтов.
- 2–5 ревьюеров (по умолчанию 3).
- Единый формат замечаний: risk-table.
- “Critical contradiction” → обязательный rework.

### Stage III — Court (AGM Verdict)
Цель: собрать финальный пакет, решить конфликты, подготовить Telegram-сообщение.
- Если требуется EXECUTE и это CRITICAL → **STOP → Owner approval**.

## 4) Жизненный цикл задачи
NEW → TRIAGED → IN_PIPELINE → IN_ROUNDTABLE → IN_COURT → (WAIT_OWNER?) → DONE → ARCHIVED

## 5) Уровни критичности
- **CRITICAL**: деньги/контракты/публичные обещания/прод-изменения/PII/удаления.
- **HIGH**: архитектура, безопасность, изменения влияющие на доход/репутацию.
- **MEDIUM**: контент-план, UX, гипотезы роста.
- **LOW**: черновики, списки, генерация идей.

## 6) Политика качества (критерии приемки)
Всегда:
1) Deliverables + ссылки/файлы
2) Чек-лист приемки
3) Журнал решений (decision log)
4) Риски и митигации
5) “Что осталось неопределенным” + варианты A/B (если нужно)

## 7) Коммуникации
Owner → Telegram → Gateway → AGM → Agents → AGM → Telegram.

## 8) Лимиты по умолчанию
- Pipeline steps: 4 (HIGH/CRITICAL: до 6)
- Roundtable reviewers: 3
- Rework cycles: 2 (3-й → эскалация Owner с вариантами)
- Telegram summary: до 1200 символов + файл отчёта при необходимости
